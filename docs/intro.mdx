# ZotFlow Documentation

## What is ZotFlow?

ZotFlow is an Obsidian plugin that eliminates the tool switching between "literature reading" and "note writing." It integrates Zotero's item browsing, PDF/EPUB annotation, citation insertion, and note management entirely within Obsidian, allowing you to complete the full read-to-write loop in a single editor, with one set of keyboard shortcuts, under one theme.

The core philosophy: **Zotero is the source of truth for bibliographic metadata, and Obsidian is your thinking and writing home base.** ZotFlow builds a bidirectional sync channel between the two — metadata, attachments, annotations, and child notes can all be pushed/pulled on demand — while using the Source Note auto-generation mechanism to create a traceable, linkable "source-of-fact layer" for each bibliographic item in your vault.

## Terminology

- **Source Note**: An auto-generated Markdown note for each Zotero item, rendered by a LiquidJS template. Defaults to `zotflow-locked: true`, read-only in Reading View. Serves as the "source-of-fact layer" in your knowledge graph.
- **Item Note**: A Zotero native child note attached to an item, which can be created/edited/deleted within ZotFlow and bidirectionally synced back to Zotero. A completely different concept from Source Note.
- **Library Reader**: The reader mode for Zotero library attachments, where annotations can be written back to Zotero via bidirectional sync.
- **Local Reader**: The reader mode for local PDF/EPUB/HTML files within your vault, where annotations are written to a co-located `.zf.json` sidecar file and never touch Zotero.
- **Editable Region**: A section within a Source Note wrapped by `<!-- ZF_NOTE_BEG_<key> -->` / `<!-- ZF_ANNO_BEG_<key> -->` markers that can be unlocked for editing.
